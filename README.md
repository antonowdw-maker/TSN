# Telegram-ассистент ТСН (демо)

Бот для жителей ТСН: ответы на частые вопросы, контакты, диспетчерская и график работы.

> В репозитории **демонстрационные данные** (`data/*.json`). Для реального ТСН замените их на актуальные. Токены — только в `.env`, не в git.

## Меню (`/start`)

| Раздел | Что внутри |
|--------|------------|
| 📋 Популярные вопросы | FAQ по категориям |
| 📞 Контакты | Руководство и офис |
| 🚨 Диспетчерская | Телефоны и режим работы |
| 📅 График работы | Расписание сотрудников |

После ответов и в разделах — кнопка **«◀️ Главное меню»**.  
Любой вопрос можно написать текстом — бот ищет ответ в FAQ.

## Команды

| Команда | Описание |
|---------|----------|
| `/start` | Приветствие и меню |
| `/help` | Справка |
| `/faq` | Популярные вопросы |
| `/contacts` | Контакты |
| `/dispatcher` | Диспетчерская |
| `/schedule` | График работы |

## Данные

- `data/faq.json` — вопросы и ответы
- `data/organization.json` — контакты, телефоны, графики

## Разработка

```bash
pip install -r requirements.txt
copy .env.example .env   # TELEGRAM_BOT_TOKEN от @BotFather
pytest -v
python -m bot.main
```

На домашнем ПК часто бывает `TimedOut` к `api.telegram.org` — для проверки в Telegram используйте деплой в облако (Railway).

Опционально: `OPENAI_API_KEY` — ответы на вопросы вне FAQ.

## Деплой

Push в `main` → GitHub Actions деплоит на **Railway** (см. `.github/workflows/deploy-railway.yml`).

Секреты в GitHub: `RAILWAY_TOKEN`, `RAILWAY_SERVICE_ID`, `TELEGRAM_BOT_TOKEN`.

Ручной чек-лист: `tests/TEST_CASES.md`.
